# 管理員功能使用指南

## 📋 功能說明

添加了一個**管理員只讀模式**，讓管理員帳號可以查看所有用戶的任務，但無法修改。

## 🔐 登入憑證

已自動創建管理員帳號：
- **用戶名**: `admin`
- **密碼**: `admin123`
- **角色**: `admin`

## 📺 管理員界面

### 特點：
1. **查看所有人的任務** - 顯示所有用戶在所選日期的任務
2. **任務前顯示人員名稱** - 格式：`👤 用戶名 - 任務標題`
3. **禁用編輯功能**：
   - ❌ 不能新建任務
   - ❌ 不能拖拽調整任務順序或狀態
   - ❌ 不能編輯任務信息
   - ❌ 不能管理欄位
   - ❌ 不能查看異動記錄（歷史）
   - ❌ 不能啟用「常態展開」功能

4. **可以執行的操作**：
   - ✅ 查看任務詳情（點擊展開）
   - ✅ 查看任務描述和圖片
   - ✅ 查看任務的工時（⏱️）
   - ✅ 切換日/周/月視圖
   - ✅ 更改密碼

## 🎯 使用場景

### 經理/主管視角
- 可以看到團隊每個成員每天的工作進度
- 按日期、週或月份查看整體工作狀況
- 了解每個任務的詳細信息和進展

### 數據分析
- 查詢特定日期的全部任務
- 統計團隊工作量分佈

## 🔄 工作流程

1. **用普通帳號登入** → 可以編輯自己的任務
2. **用管理員帳號登入** → 查看所有人的任務（只讀模式）

## 💾 數據庫更改

已執行以下更改：

### 1. 表結構修改
```sql
ALTER TABLE users ADD COLUMN role VARCHAR(20) DEFAULT 'user';
ALTER TABLE users ADD INDEX idx_role (role);
```

### 2. 新增管理員帳號
```sql
INSERT INTO users (username, email, password_hash, role)
VALUES ('admin', 'admin@example.com', '$2y$10$...', 'admin');
```

## 🔧 技術實現細節

### 後端 (PHP)
- **auth.php**: 登入時返回 `role` 字段
- **tasks.php**: 管理員查詢時返回所有用戶的任務 + `username`

### 前端 (Vue 3)
- **userStore.js**: 保存登入用戶的 `role` 信息
- **taskStore.js**: 添加 `isAdmin` 標誌，判斷是否為管理員
- **DashboardView.vue**:
  - 顯示任務時檢查 `task.username`
  - 禁用拖拽：`:draggable="!taskStore.isAdmin"`
  - 隱藏編輯按鈕：`v-if="!taskStore.isAdmin"`
  - 隱藏新建按鈕：`v-if="!taskStore.isAdmin"`
  - 添加只讀樣式：CSS `.task-card.readonly`

## 📝 注意事項

1. **管理員帳號密碼請更改** - 建議登入後立即更改密碼
2. **角色不可在前端修改** - 需要直接修改數據庫
3. **權限驗證在後端** - 管理員無法通過前端欺騙後端執行修改操作

## 🧪 測試步驟

1. 啟動後端和前端
2. 用管理員帳號 (admin/admin123) 登入
3. 驗證：
   - ✓ 看到所有用戶的任務
   - ✓ 任務前有用戶名顯示
   - ✓ 無法拖拽任務
   - ✓ 編輯按鈕隱藏
   - ✓ 新建工作按鈕隱藏
   - ✓ 管理欄位按鈕隱藏
4. 切換回普通帳號，驗證功能恢復正常
